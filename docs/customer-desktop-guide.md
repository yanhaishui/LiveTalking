# 码布斯 MEH 数字人系统 客户使用手册（桌面版）

## 1. 适用范围
- 适用于 Windows 安装包（`.exe`）与 macOS 安装包（`.dmg`）。
- 本手册面向运营同学，不需要命令行。

## 2. 安装

## 2.1 Windows
1. 双击 `MEH Digital Human-<version>-win-x64.exe`。
2. 选择安装目录，点击安装。
3. 首次可能出现 SmartScreen 提示，点击“更多信息”后继续安装（正式签名后此提示会减少）。

## 2.2 macOS
1. 打开 `MEH Digital Human-<version>-mac-arm64.dmg`。
2. 将应用拖入 `Applications`。
3. 首次打开若出现安全提示，请在“系统设置 -> 隐私与安全性”中允许。

## 3. 首次启动（推荐流程）
1. 打开应用后会出现“首次启动向导”。
2. 点击“运行体检”，确认：
   - Python 可用
   - wav2lip 模型可用
   - XTTS 地址可达（如需要声音克隆）
   - 端口无冲突
3. 若有错误，按页面建议修复后再次体检。
4. 点击“我已完成，进入系统”。

## 4. 日常使用
1. 在“运行设置”里选择模式：
   - 本机模式：桌面端托管本地 `control_api`
   - 云端模式：连接远程控制 API
2. 点“启动 API”（本机模式）。
3. 左侧嵌入管理台中执行：形象管理、声音管理、脚本轮播、直播控制。
4. 开播前建议先点“运行体检”。

## 5. 常见操作
- 导出诊断包：运行设置区点击“导出诊断包”，发给技术支持。
- 导入/导出设置：用于多机器快速复制配置。
- 更新策略：
  - `stable` 给正式环境
  - `beta` 给测试环境
  - 出现异常可关闭“启用自动更新检查”作为临时回滚措施

## 6. 故障排查
- API 启动失败：通常是 9001 端口被占用或 Python 环境异常。
- 无法声音克隆：检查 XTTS 地址和 9000 端口连通。
- 视频无法生成：检查模型文件与 `data/avatars` 权限。
- 若仍异常，导出诊断包给技术支持。

## 7. 版本升级建议
1. 先在测试机验证新版本（建议 beta 通道）。
2. 验证通过后切换到 stable 通道。
3. 重要直播前 1 小时内不建议升级版本。
